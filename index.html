Main Content

Before understanding a thread, one first needs to understand a UNIX process. A process is created by the operating system, and requires a fair amount of "overhead". Processes contain information about program resources and program execution state, including: o Process ID, process group ID, user ID, and group ID o Environment o Working directory. o Program instructions o Registers o Stack o Heap o File descriptors o Signal actions o Shared libraries o Inter-process communication tools (such as message queues, pipes, semaphores, or shared memory).

Sidebar

POSIX Threads, commonly known as pthreads, isanexecution modelthat exists independently from a programming language, as well as a parallel execution model. It allows a program to control multiple different flows of work that overlap in time. The POSIX standard has continued to evolve and undergo revisions, including the Pthreads specification. Pthreads are defined as a set of C language programming types and procedure calls, implemented with a pthread.h header/include file and a thread library - though this library may be part of another library, such as libc, in some implementations.
